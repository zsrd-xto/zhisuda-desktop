> **版本**: v1.1 | **状态**: 修订待审批 | **创建日期**: 2026-06-20 | **修订日期**: 2026-06-24 | **作者**: PM Agent

## 一、产品概述

### 1.1 产品定位

职速达是一款面向求职者的 **AI 驱动的桌面端求职辅助工具**（独立 Electron 应用），通过内嵌 WebView 对接主流招聘平台（MVP 仅 Boss 直聘，v0.2 扩展 51job），实现岗位智能筛选、半自动批量投递、HR 消息提醒等功能，帮助求职者从重复劳动中解放出来，将精力集中在「决策」而非「执行」上。

### 1.2 一句话价值主张

> **"好工作，职速达——智能筛选岗位，确认即投，求职更高效。"**

> **宣传约束**：官网与营销文案不使用「全自动海投」「跑赢 90% 求职者」等易引监管关注的表述；强调「辅助工具、用户确认、本地数据」。

### 1.3 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 25-35岁互联网/IT从业者，有1-8年工作经验 |
| **用户标签** | 在职跳槽、海投期应届生、转行求职者 |
| **技术素养** | 中高（能接受桌面应用+API Key配置） |
| **核心痛点** | 每天在多平台重复投递50+岗位，回复HR消息耗时1-2小时 |
| **付费意愿** | 中高（求职是刚需，愿花¥30-100/月换取效率提升） |

#### 用户故事

**用户A - 在职跳槽的程序员小王**
> "我每天下班后花2小时在Boss直聘上投简历，投了3周才拿到2个面试。职速达帮我设好条件后自动筛选+批量投递，3天投了200个匹配岗位，面试邀请涨到8个。"

**用户B - 应届生小李**
> "我第一次找工作，不知道怎么回复HR的问题。职速达的AI帮我回复了'是否接受外包'的问题，还提醒我哪些是敏感问题需要自己回复。"

**用户C - 转行求职的小张**
> "我的简历是按原来行业写的，职速达的AI简历优化功能帮我用目标行业的语言重写了项目经历，投递回复率从5%提高到20%。"

## 三、功能详细描述

### 3.1 模块一：账号系统

#### 3.1.1 本地账号（MVP v0.1）

**功能描述**：首次启动自动创建设备级本地用户，可选设置昵称；所有数据存储在本地 SQLite，无需联网注册。

**验收标准**：
```gherkin
Given 用户首次打开职速达应用
When 应用完成初始化
Then 自动创建本地用户（device_id），进入主界面

Given 用户在个人中心
When 设置昵称并保存
Then 昵称本地持久化，下次启动仍显示

Given 用户在设置页点击「清除所有数据」
When 确认操作
Then 删除本地 SQLite 全部数据并重新初始化
```

**非功能需求**：
- 本地用户 ID 使用 UUID，与设备绑定
- 无需后端服务，MVP 零服务器账号成本

#### 3.1.2 手机号注册/登录（v0.2）

**功能描述**：v0.2 引入手机号+验证码注册，用于订阅鉴权、多设备同步（不含招聘平台 Cookie/简历上传云端）。

**验收标准**：
```gherkin
Given 用户打开职速达应用（v0.2+）
When 输入手机号并获取验证码
Then 发送短信验证码（60秒倒计时）

Given 用户输入正确的手机号和验证码
When 点击登录
Then 登录成功，本地数据可选择绑定云端账号
```

**非功能需求**：
- 验证码有效期：5分钟
- 登录态保持：7天免登录
- 密码加密存储：bcrypt（邮箱密码登录为可选）

### 3.2 模块二：简历管理

#### 3.2.1 简历上传

**功能描述**：支持上传PDF/Word格式简历，系统自动解析为结构化数据。

**验收标准**：
```gherkin
Given 用户在简历管理页面
When 上传一份PDF/word格式简历
Then 系统解析简历内容，展示结构化预览（基本信息/个人优势/工作经历/项目介绍/教育经历/证书和荣誉）

Given 解析结果有误
When 用户点击"编辑修正"
Then 进入在线编辑器，可逐字段修改

Given 用户上传非PDF/Word格式文件
When 点击上传
Then 提示"仅支持PDF和Word格式"
```

**解析字段映射**：
| 字段 | 来源 | 必填 |
|------|------|------|
| 姓名 | 简历头部 | ✅ |
| 手机号 | 简历头部 | ✅ |
| 邮箱 | 简历头部 | ✅ |
| 求职意向 | 简历头部 | ✅ |
| 工作经历 | 公司/职位/时间/描述 | ✅ |
| 项目经历 | 项目名/角色/描述 | ✅ |
| 教育背景 | 学校/专业/学历/时间 | ✅ |
| 技能标签 | 技能列表 | ⬜ |
| 自我评价 | 个人优势 | ⬜ |

#### 3.2.2 在线编辑

**功能描述**：解析后提供表单化的在线编辑器，支持逐字段修改。

**交互要求**：
- 左侧结构化表单，右侧实时预览简历效果
- 支持拖拽排序（工作经历、项目经历的顺序）
- 自动保存（每30秒或字段变更时）
- 支持导出为PDF/Word

### 3.3 模块三：求职偏好设置

**功能描述**：用户可配置**多条**求职偏好，每条为「一个目标岗位 + 一个城市 + 薪资区间」，名称自动生成（如 `AI应用开发·深圳·20-25K`）。偏好用于 Boss 岗位搜索抓取与后续规则匹配。

> **v0.1 实现说明**：MVP 每条偏好为单岗位/单城市组合；多选标签留待后续版本扩展。

**配置项**（单条偏好）：

| 条件 | 类型 | 说明 | 必填 |
|------|------|------|------|
| 目标岗位 | 文本 | 如：AI应用开发、前端开发 | ✅ |
| 目标城市 | 文本 | 如：深圳、北京 | ✅ |
| 薪资范围 | 区间 | 如：20K-25K | ✅ |
| 行业偏好 | 多选 | 如：互联网、金融、教育 | ⬜ |
| 公司规模 | 多选 | 如：100-500人、上市公司 | ⬜ |
| 五险一金 | 开关 | 必须有五险一金 | ⬜ |
| 双休要求 | 开关 | 必须双休 | ⬜ |
| 排除外包 | 开关 | 排除外包岗位 | ⬜ |
| 黑名单公司 | 列表 | 不投递的公司名称 | ⬜ |
| 排除关键词 | 列表 | 岗位描述含这些词则排除 | ⬜ |

**验收标准**：
```gherkin
Given 用户设置了"目标城市=北京，薪资>=20K，排除外包=是"
When 系统筛选岗位
Then 仅展示北京地区、薪资20K以上、非外包的匹配岗位

Given 用户添加"某公司"到黑名单
When 系统抓取到该公司的岗位
Then 自动过滤，不展示也不投递
```

### 3.4 模块四：自动投递（核心功能）

#### 3.4.1 平台登录（WebView）

**功能描述**：应用内嵌WebView，用户在应用内登录招聘平台。

**交互流程**：
```
用户选择平台（Boss直聘/51job）
    ↓
WebView加载平台登录页
    ↓
用户手动登录（输入账号密码/扫码）
    ↓
登录成功，WebView保存Cookie
    ↓
系统注入自动化脚本（控制台不可见）
    ↓
后续操作通过脚本自动执行
```

**技术要求**：
- WebView 需模拟正常浏览器 UA 和行为特征
- 登录态本地加密存储，不上传云端
- MVP 仅支持 Boss 直聘；v0.2 扩展 51job
- Cookie 过期时自动提示用户重新登录
- 自动化失败时支持 **人工接管**：用户可在 WebView 内手动完成操作

#### 3.4.2 岗位抓取与匹配

**功能描述**：从已登录的招聘平台抓取岗位列表，根据用户偏好条件进行匹配评分。

**匹配算法（MVP v0.1 — 规则引擎）**：
```
匹配度评分 = 薪资匹配分(40%) + 岗位名称匹配分(25%) + 公司匹配分(20%) + 城市匹配分(10%) + 关键词匹配分(5%)

薪资匹配分 = 岗位薪资范围与用户期望的重叠度
岗位名称匹配分 = 关键词命中（目标岗位标签与岗位名称 Jaccard 相似度）
城市匹配分 = 精确匹配=100，同省=60，其他=0
公司匹配分 = 扣分制（黑名单扣100，外包公司扣50，不标明五险一金扣5，不标明双休扣10）
关键词匹配分 = 技能标签/排除关键词命中情况
```

> **v0.1 实现说明（当前代码）**：筛选与打分已分离，与上文加权公式并存于 PRD 规划、实现以本节为准。
>
> **筛选（通过/不通过）**：岗位名 Jaccard ≥ 名称匹配阈值（默认 20%，偏好可配）；岗位薪资下限 ≥ 抓取薪资下限；黑名单公司与排除关键词命中则过滤；城市不参与筛选（搜索 API 已限定）。
>
> **打分（累加积分，按总分降序）**：岗位职责关键词每命中 +10 分；公司规模/注册资本/福利（五险一金、双休、包吃包住）按档位加减分。
>
> **抓取**：Boss 搜索 API 传 `query` + 城市 code；岗位页可覆盖岗位名、城市、薪资下限，写入批次 `conditions_json`。

**匹配算法（v0.2 — 向量语义增强）**：
```
在规则引擎基础上增加：
岗位职责匹配分 = 语义相似度（Cosine，基于 embedding API + 用户自带 Key）
```

**数据抓取策略**：
- 抓取触发：用户手动触发，须先选择一条求职偏好（MVP）
- 抓取接口：Boss 搜索 API（`query` + 城市 code），客户端按偏好过滤
- 结果存储：按「抓取时间 + 条件快照」写入批次表，保留 7 天
- 单次抓取上限：每平台最多 100 个岗位（MVP）
- 请求间隔：随机 5-15 秒（模拟人工浏览）
- 异常处理：抓取失败时记录日志，提示用户人工接管

**验收标准**：
```gherkin
Given 用户设置偏好为"前端开发，北京，20K-30K"
When 系统完成岗位抓取
Then 展示匹配岗位列表，按匹配度评分降序排列
And 每个岗位显示：公司名、岗位名、薪资、城市、是否外包、单/双休息、得分

Given 用户设置抓取薪资下限 20K，某岗位薪资为 18-22K
When 执行筛选
Then 该岗位不通过（岗位薪资下限 18K < 20K）
```

#### 3.4.3 批量投递（L1模式）

**功能描述**：用户审核匹配结果后，批量投递选中的岗位。

**交互流程**：
```
系统展示匹配岗位列表
    ↓
用户操作：
  ☑ 勾选要投递的岗位
  ☐ 取消不想投的岗位
  ⭐ 标记重点关注
  🚫 加入黑名单
    ↓
用户点击"批量投递"
    ↓
系统逐个执行投递（间隔15-30秒随机，单次最多10个岗位）
    ↓
投递失败时提示人工接管（WebView 内手动投递）
    ↓
实时展示投递进度和结果
    ↓
投递完成，更新数据看板
```

**投递状态机**：
```
待投递 → 投递中 → 已投递 → 已查看 → 已回复 → 面试邀请
                  ↘ 投递失败（平台限制/网络错误）
```

#### 3.4.4 自动投递（L2模式，v0.2）

> **MVP 不包含**：v0.1 个人中心可展示「定时任务即将推出」占位，不实现调度逻辑。

**功能描述**：配置定时任务后，系统按计划自动执行"抓取→匹配→投递"全流程。

**定时任务配置项**：

| 配置 | 说明 | 默认值 |
|------|------|--------|
| 触发平台 | Boss直聘/51job/全部 | 全部 |
| 执行频率 | 每天N次 | 每天1次 |
| 执行时间 | 具体时间点 | 09:00 |
| 每次投递上限 | 最多投递几个岗位 | 20 |
| 是否需要确认 | L1(需确认) / L2(自动) | L1 |
| 运行周期 | 起止日期 | 手动停止 |

**验收标准**：
```gherkin
Given 用户配置定时任务：每天09:00，Boss直聘，自动模式，上限20
When 到达触发时间
Then 系统自动抓取岗位、匹配条件、执行投递

Given 已达到当天投递上限20个
When 还有匹配岗位未投递
Then 停止投递，记录剩余岗位到"待投递队列"

Given 投递过程中遇到平台反爬限制
When 连续3次投递失败
Then 暂停该平台任务，发送通知提醒用户
```

### 3.5 模块五：HR消息智能回复

#### 3.5.1 消息监控（MVP：仅提醒）

**功能描述**：
- **MVP v0.1**：通过 WebView 定期检查 HR 新消息，发送桌面通知，不自动回复
- **v0.2+**：按策略分类并自动/AI 回复

**消息轮询策略（MVP）**：
- 频率：每 5-10 分钟随机检查一次
- 仅检查未读消息
- 检测到新消息 → 桌面通知 + 消息列表展示

#### 3.5.2 消息分类与回复策略（v0.2）

> MVP 不实现自动回复，以下策略于 v0.2 启用。

**分类规则引擎**：

| 分类 | 触发关键词/条件 | 系统行为 | 优先级 |
|------|-----------------|----------|--------|
| **打招呼** | "在吗"、"你好"、"看了你的简历" | 自动回复预设话术 | 自动 |
| **索要简历** | "简历"、"发一份简历"、"CV" | 自动发送简历附件 | 自动 |
| **常规咨询** | "是否接受外包/出差/加班" | AI生成回复 | 自动 |
| **薪资询问** | "期望薪资"、"薪资范围" | 按用户预设回复 | 自动 |
| **面试邀请** | "面试"、"约个时间"、"来公司聊聊" | 🔔 通知提醒，不自动回复 | 手动 |
| **隐私问题** | "年龄"、"婚育"、"结婚"、"生育" | 🔔 通知提醒，标记敏感 | 手动 |
| **无法分类** | 不匹配任何规则 | 🔔 通知提醒 | 手动 |

**自动回复话术模板**：

```yaml
# 用户可自定义的回复模板
greeting:
  - "您好！感谢您的关注，请问有什么可以帮您的？"
  - "在的，请问贵司这个岗位的具体情况是？"

resume:
  - "好的，简历已发送，请查收。"
  # 系统自动附加简历附件

outsource:
  - "感谢您的介绍。目前更倾向于甲方正式岗位，如果有合适的机会希望能进一步沟通。"
  # 或由AI根据用户预设生成

salary:
  - "期望薪资{salary_range}，具体可根据岗位情况面议。"
  # 取用户预设的薪资范围
```

**验收标准**：
```gherkin
Given HR发送消息"在吗"
When 系统检测到该消息
Then 自动回复预设话术"您好！感谢您的关注，请问有什么可以帮您的？"

Given HR发送消息"明天下午2点方便面试吗"
When 系统检测到该消息
Then 发送桌面通知"【Boss直聘】XX公司HR邀请你面试"
And 不自动回复
And 在消息列表标记为"待处理"

Given HR发送消息"你多大了？结婚了吗？"
When 系统检测到该消息
Then 发送通知"有HR询问隐私信息"
And 标记为"敏感消息"
And 不自动回复

Given 用户配置了OpenAI API Key
When 遇到"是否接受外包"类问题
Then 调用AI根据用户预设的"不接受外包"生成自然语言回复
And 将回复内容记录到日志
```

### 3.6 模块六：AI简历优化（v0.3）

```
    - 逐段工作经历

    - 量化成果建议
    ↓
Step 4: 项目经历优化
    - 逐个项目
    - AI建议突出技术栈和成果
    ↓
Step 5: 技能标签优化
    - 根据目标岗位推荐热门技能
    - 去除过时技能
    ↓
Step 6: 预览 & 导出
```

## 四、非功能性需求

### 4.1 性能

| 指标 | 要求 |
|------|------|
| 应用启动时间 | < 3秒（冷启动） |
| 岗位抓取响应 | 单次抓取 < 30秒 |
| 投递操作延迟 | 单次投递 < 5秒（不含平台响应） |
| AI回复延迟 | < 3秒（不含模型推理时间） |
| 内存占用 | < 500MB（含WebView） |

### 4.2 安全

| 措施 | 说明 |
|------|------|
| 数据加密 | 用户密码bcrypt加密，API Key AES-256加密存储 |
| 本地存储优先 | 简历、聊天记录等敏感数据仅存储在本地 |
| 通信加密 | 所有网络请求使用HTTPS |
| API Key安全 | 用户API Key仅在本地使用，不上传到职速达服务器 |
| 登录态安全 | 招聘平台Cookie本地加密存储 |

### 4.3 可靠性

| 指标 | 要求 |
|------|------|
| 自动重试 | 网络请求失败自动重试3次 |
| 断点续传 | 投递中断后可从上次位置继续 |
| 日志记录 | 所有操作记录完整日志，便于排查问题 |
| 崩溃恢复 | 异常退出后恢复未完成的任务状态 |

### 4.4 合规

| 要求 | 说明 |
|------|------|
| 用户协议 | 明确告知用户产品仅为辅助工具，用户自行承担使用风险（含封号） |
| 隐私政策 | 不收集、不存储用户的招聘平台账号密码 |
| 操作频率 | 严格控制操作频率，MVP 单次批量投递 ≤10，间隔 15-30 秒 |
| 数据删除 | 用户可随时删除所有本地数据 |

### 4.5 分发与更新

| 要求 | 说明 |
|------|------|
| 产品官网 | 独立官网提供 Windows/macOS 下载入口（对标 OpenClaw 桌面版官网结构） |
| 自动更新 | electron-updater，启动时检查，支持跳过版本 |

### 6.2 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 桌面框架 | Electron 28+ | 成熟稳定，WebView支持好 |
| 前端框架 | React 18 + TypeScript | 生态丰富，类型安全 |
| UI组件库 | Ant Design 5 | 企业级组件库，开箱即用 |
| 状态管理 | Zustand | 轻量，适合中小型应用 |
| 本地数据库 | SQLite (better-sqlite3) | 轻量、无需额外服务 |
| WebView自动化 | Electron webContents + PlatformAdapter 脚本 | 通过 executeJavaScript / preload 桥接控制 BrowserView，非 Puppeteer |
| 平台适配层 | PlatformAdapter 接口 | Boss DOM 变更时仅修改适配器，隔离平台差异 |
| 自动更新 | electron-updater | 启动时检查官网 latest.yml，支持跳过版本 |
| 崩溃上报 | Sentry（可选关闭） | 匿名崩溃日志，默认关闭需用户同意 |
| 简历解析 | pdf-parse + mammoth | PDF/Word解析 |
| AI调用 | OpenAI SDK (兼容多提供商) | 统一接口，用户自带Key |
| 打包分发 | electron-builder | 支持Win/Mac/Linux |
| 定时任务 | node-cron | 轻量级任务调度 |
